建议

原则
技术栈自选、能启动、能调试、能提交、能复现，比盲目追求新技术更重要。

科研报国  当为人先
后端接口怎么设
计？
接口是前后端协作的“契约”：输
入、处理、输出必须清晰
接口 方法 输入 输出 页面用途
/api/protocols GET 无 协议列表 导航页
/api/dns/resolve POST domain 步骤+缓存+日志 DNS页
/api/arp/table GET 无 ARP表项 ARP页
/api/network/simulate POST url 完整事件流 综合场景
/api/logs GET sessionId 日志列表 日志面板
{} JSON 返回建议
{ code, message, data, steps, logs }：保持结构统一，便于前端处理成功、失败、过程步骤和日志。

主流数据库怎么选？
从轻量演示到完整工程，数据库选择可以逐级提升
PostgreSQL / MongoDB
复杂数据、知识图谱、进阶项目
MySQL / SQLite
表结构清晰，适合课程项目
LocalStorage / JSON
纯前端演示，适合快速原型
A 入门建议
先用 JSON / LocalStorage 完成演示，再迁移到 
SQLite 或 MySQL。
B 标准建议
协议表、日志表、设备表使用关系型数据
库最清晰。
C 进阶建议
知识图谱、问答索引或半结构化数据可考虑 
MongoDB。
推荐开发工具：Navicat

===== PAGE 16 =====
科研报国  当为人先
Git 和 GitHub 的区别
Git 是本地版本控制工具，GitHub 是远程代码托管平台
G Git：本地工具
安装在电脑上
记录代码版本
执行 add / commit / branch
可以回退与比较修改
GH GitHub：远程平台
在线保存代码仓库
展示提交记录
支持协作与项目管理
便于教师检查过程
push / pull
关系：Git 管理本地代码，GitHub 保存远程仓库，两者配合完成项目管理。
